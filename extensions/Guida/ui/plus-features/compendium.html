Compendium

Pannello per consultare compendi installati (regole, mostri, incantesimi, oggetti, background, ecc.) con navigazione ad albero, ricerca, tag e markdown arricchito. È pensato per il Master e, dove configurato, anche per i giocatori con contenuti visibili.

Installazione e aggiornamento contenuti

Dal cruscotto o dai flussi di import puoi caricare JSON e/o archivi ZIP che contengono uno o più compendi. Ogni installazione aggiunge alberi di collezioni e voci (slug univoci per voce). Puoi tenere più compendi (es. SRD 5e + homebrew) e passare dall’uno all’altro.

Navigazione

  • Sidebar: compendio → collezioni annidate (anche gerarchiche) → voci.
  • Breadcrumb: salta rapidamente a un livello superiore senza perdere il contesto.
  • Ricerca globale: filtra per nome tra le voci caricate (utile in sessione quando il nome è noto).
  • Tag (se presenti nei dati): filtri per categoria, tipo, livello incantesimo, ecc.

Visualizzazione voce

Il testo è solitamente Markdown: titoli, liste, tabelle e blocchi codice. I link interni al compendio (sintassi dedicata) si aprono dentro l’estensione. Le immagini remote o interne vengono renderizzate nel viewer.

Condivisione in chat

Puoi inviare un estratto della voce (o un riassunto formattato) nella chat di gioco così tutti vedono lo stesso riferimento senza aprire il PDF sul proprio PC.

Collegamenti nelle note (deep link)

Nelle note di PlanarAlly puoi usare collegamenti del tipo [testo visibile](qe:compendium/collection/…) (la forma esatta dipende dal formato del tuo database) così, cliccando dalla nota, si apre la voce giusta nel compendio.

Immagini e libreria asset

Da una voce, con tasto destro sull’immagine puoi aggiungere l’illustrazione agli asset della campagna. Il file viene salvato in una cartella deterministica (percorso extensions/compendium/… con sottocartella per slug voce per evitare collisioni tra voci omonime). Dopo il salvataggio puoi trascinare l’immagine sulla mappa come qualsiasi altro asset; se la ricarichi in futuro, il client può risolvere l’hash senza duplicare file.

Traduzioni e contesto

Se usi database multilingua (es. contenuti IT/EN), l’interfaccia può adattarsi al locale della sessione dove previsto dai metadati del compendio.

Compendium

Browse installed compendiums (rules, monsters, spells, items, backgrounds, etc.) with a tree, search, tags, and rich Markdown. Built for the GM, and—where configured—for players with visible content.

Installing content

Import JSON and/or ZIP packs from the dashboard. Each install adds collections and entries (per-entry slugs). You can keep multiple compendiums side by side (e.g. SRD 5e + homebrew).

Navigation

  • Sidebar: compendium → nested collections → entries.
  • Breadcrumb for quick jumps up the hierarchy.
  • Global search filters by name across loaded entries.
  • Tags (when present in data) for category filters, spell level, etc.

Entry view

Entries render as Markdown: headings, lists, tables, code blocks. Internal compendium links open inside the extension. Images render inline.

Share to chat

Push a snippet or formatted summary to game chat so everyone shares the same reference.

Deep links in notes

Use [label](qe:…)-style links in PlanarAlly notes so clicks jump straight into the right entry (exact format depends on your database).

Images and assets

Right-click an image in an entry to add it to the campaign library under a deterministic path (extensions/compendium/…, including a per–entry slug folder to avoid name clashes). Then drag it to the map like any asset; on reload the client can resolve the same file hash without duplicates.

Locales

Multilingual databases can respect session locale when metadata provides it.
