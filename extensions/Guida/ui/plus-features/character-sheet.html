Scheda del personaggio

Scheda D&D 5e (layout “italiano v2”) con due tab principali: scheda completa e incantesimi / storia. I dati sono salvati lato server per campagna; il Master ha visione completa, i giocatori lavorano sulle proprie schede (e su quelle rese visibili).

Struttura dell’interfaccia

  • Barra superiore: ricerca nell’elenco schede, creazione nuova scheda, esportazione/importazione JSON, stampa PDF, e quando applicabile anteprima “come giocatore”.
  • Sidebar: elenco schede con nome; clic per caricare. Seleziona una scheda prima di modificarla — il pulsante Abilita modifiche / Solo lettura compare solo quando una scheda è caricata e hai permesso di edit.
  • Tab interni: passa tra scheda principale (statistiche, attacchi, equipaggiamento, tratti) e sezione incantesimi/storia.

Modifica e salvataggio

Attiva la modalità modifica per editare campi. Molti valori si salvano automaticamente dopo le modifiche (debounce). In sola lettura i campi sono disattivati ma i link al compendio restano cliccabili per consultazione.

Collegamento a personaggio e token

Puoi associare una scheda a un personaggio della campagna. Dal menu contestuale del segnalino (token) puoi aprire direttamente la scheda collegata senza cercarla nella lista.

Compendium

Nei campi testo supportati, i riferimenti a contenuti del compendio (quinta edizione) permettono di inserire incantesimi e regole coerenti con i database installati. I link si comportano come nel resto del client PlanarAlly.

Import / Export

  • Esporta JSON: backup dello stato completo della scheda.
  • Importa JSON: crea una nuova scheda o aggiorna dati (utile per PG generati dall’AI Generator o migrati da file).
  • Stampa: apre una finestra di stampa con il layout formattato per carta.

Visibilità e permessi

Il Master può nascondere o mostrare schede ai giocatori (icona visibilità). I giocatori vedono le proprie schede e quelle rese esplicitamente visibili. Le operazioni di eliminazione sono limitate ai permessi previsti dal backend.

Character Sheet

D&D 5e-style sheet with two tabs: main sheet and spells & backstory. Data is stored per campaign on the server; the GM sees everything, players edit their own sheets (and any shared ones).

UI layout

  • Top bar: search, new sheet, JSON import/export, print, and (for the GM) a player-preview toggle when available.
  • Sidebar: sheet list. Load a sheet before editing—the Enable edits / Read-only toggle only appears when a sheet is loaded and you have edit rights.
  • Inner tabs: switch between main stats and spells/story.

Editing and saving

Enable edit mode to change fields. Many values auto-save after debounced changes. In read-only mode inputs are disabled but compendium links stay clickable.

Character and token link

Bind a sheet to a campaign character. From the token context menu, open the linked sheet without hunting the list.

Compendium

Supported text fields accept compendium links so spells and rules match your installed databases.

Import / Export

  • Export JSON for backups.
  • Import JSON to create or hydrate sheets (AI Generator output, migrations).
  • Print opens a print-friendly layout.

Visibility

The GM can toggle visibility to players per sheet. Deletes respect server-side permissions.
